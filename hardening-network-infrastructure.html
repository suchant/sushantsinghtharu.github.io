Requirements
Note: Not all security features will be configured on all devices,
although they normally would be in a production network.
Configure Basic Router Security
Configure the following on R1:
o Minimum password length is 10 characters.**
R1(config)#security passwords min-length 10
o Encrypt plaintext passwords.**
R1(config)#service password-encryption
o Privileged EXEC mode secret password is ciscoenapa55.**
R1(config)#enable secret ciscoenapa55
o Console line password is ciscoconpa55, timeout is 15
minutes, and console messages should not interrupt command entry.
R1(config)#line console 0
R1(config-line)#password ciscoconpa55
R1(config-line)#logging synchronous
o A message-of-the-day (MOTD) banner should include the
word unauthorized.
R1(config)#banner motd 'unauthorized'

Configure Basic Switch Security
Configure the following on S1:
o Encrypt plaintext passwords.
S1(config)#service password-encryption
o Privileged EXEC mode secret password is ciscoenapa55.
S1(config)#enable secret ciscoenapa55
o Console line password is ciscoconpa55, timeout is 5
minutes, and consoles messages should not interrupt command entry.
S1(config)#line console 0
S1(config-line)#password ciscoconpa55
S1(config-line)#exec-timeout 5
S1(config-line)#logging synchronous
o Password for the vty lines is ciscovtypa55, timeout is 5
minutes, and login is required.
S1(config)#line vty 0 4
S1(config-line)#password ciscovtypa55
S1(config-line)#exec-timeout 5
S1(config-line)#login
o A MOTD banner should include the word unauthorized.
S1(config)#banner motd 'unauthorized'
Configure trunking between S1 and S2 with the following settings:
o Set the mode to trunk and assign VLAN 99 as the native
VLAN.
S1(config-if)#switchport trunk native vlan 99
o Disable the generation of DTP frames.
S1(config-if)#switchport mode trunk
S1(config-if)#switchport nonegotiate
o Enable storm control for broadcasts to a 50 percent
suppression level.
S1(config-if)#storm-control broadcast level 50.0
Configure the S1 with the following port settings:
o Fa0/6 should only allow access mode, set to PortFast,
and enable BPDU guard.
S1(config)#interface fastEthernet 0/6
S1(config-if)#switchport mode access
S1(config-if)#spanning-tree portfast
S1(config-if)#spanning-tree bpduguard enable
o Fa0/6 uses basic default port security with dynamically
learned MAC addresses added to the running configuration.
S1(config-if)#switchport port-security
S1(config-if)#switchport port-security mac-address
sticky
o All other ports should be disabled.
Note: Although not all ports are checked, your
instructor may want to verify that all unused ports are disabled.

. Configure SSH
Configure the following on R1:
Note: The RSA key is already generated.
o The domain name is ccnasecurity.com
R1(config)#ip domain name ccnasecurity.com
o The RSA key should be generated with a 1024 modulus bits.
R1(config)#crypto key generate rsa general-keys modulus 1024
o Only SSH version 2 is allowed.
R1(config)#ip ssh version 2
o Only SSH is allowed on vty lines.
R1(config)#line vty 0 4
R1(config-line)#transport input ssh
Verify that PC-C can remotely access R1 (209.165.200.233) using
SSH.
Secure Against Login Attacks
Configure the following on R1:
o If a user fails to log in twice within a 30-second time span, then
disable logins for one minute.
R1(config)#login block-for 60 attempts 2 within 30
o Log all failed login attempts.
R1(config)#login on-failure log
R1#sh login failures

Configure Site-to-Site IPsec VPNs
Note: Some VPN configurations are not scored. However, you should be able
to verify connectivity across the IPsec VPN tunnel.
Configure the following on R1:
o Create an access-list to identify interesting traffic on R1.
Configure ACL 101 to allow traffic from the R1 Lo1 network
to the R3 Fa0/1 LAN.
R1(config)#access-list 101 permit ip 172.20.1.0
0.0.0.255 172.30.3.0 0.0.0.255
Explicitly deny all other traffic.
R1(config)#access-list 102 deny ip any any
o Configure the crypto isakmp policy 10 Phase 1 properties on R1
along with the shared crypto key ciscovpnpa55. Use the following
parameters:
Key distribution method: ISAKMP
R1(config)#crypto isakmp policy 10
Encryption: aes 256
R1(config-isakmp)#encryption aes 256
Hash: sha-1
R1(config-isakmp)#hash sha
Authentication method: pre-shared
R1(config-isakmp)#authentication pre-share
Key exchange: DH Group 5
R1(config-isakmp)#group 5
IKE SA lifetime: 3600
R1(config-isakmp)#lifetime 3600
ISAKMP key: ciscovpnpa55
R1(config)#crypto isakmp key ciscovpnpa55 address
10.20.20.1
R1#sh crypto isakmp policy
o Create the transform set VPN-SET to use esp-aes 256 and esp-
sha-hmac. Then create the crypto map CMAP that binds all of the Phase 2
parameters together. Use sequence number 10 and identify it as an ipsec-
isakmp map. Use the following parameters:
Transform Set: VPN-SET
Transform Encryption: esp-aes 256
Transform Authentication: esp-sha-hmac
Perfect Forward Secrecy (PFS): group5
Crypto Map name: CMAP
SA Establishment: ipsec-isakmp
R1(config)#crypto ipsec transform-set VPN-SET esp-aes
256 esp-sha-hmac
R1(config)#crypto map CMAP 10 ipsec-isakmp
R1(config-crypto-map)#set transform-set VPN-SET
R1(config-crypto-map)#set pfs group5
R1(config-crypto-map)#set peer 10.20.20.1
R1(config-crypto-map)#match address 101
o Bind the crypto map CMAP to the outgoing interface.
R1(config)#interface serial 0/0/0
R1(config-if)#crypto map CMAP
Repeat the site-to-site VPN configurations on R3 so that they mirror
all configurations from R1.
Ping the Lo1 interface (172.20.1.1) on R1 from PC-C. Then on R3, use
the show crypto ipsec sa command to verify the number of packets is more
than 0, indicating that the IPsec VPN tunnel is working.

Configure Firewall and IPS Settings
Configure a ZPF on R3 using the following requirements:
o Create zones named IN-ZONE and OUT-ZONE.
R3(config)#zone security IN-ZONE
R3(config)#zone security OUT-ZONE
o Create an ACL number 110 that defines internal traffic,
permitting all IP protocols from the 172.30.3.0/24 source network to any
destination. Explicitly deny all other traffic.
R3(config)#access-list 110 permit ip 172.30.3.0 0.0.0.255 any
o Create a class map named INTERNAL-CLASS-MAP that uses the
match-all option and ACL 110.
R3(config)#class-map type inspect match-all INTERNAL-CLASS-
MAP
R3(config-cmap)#match access-group 110
o Create a policy map named IN-2-OUT-PMAP that uses the class
map INTERNAL-CLASS-MAP to inspect all matched traffic.
R3(config)#policy-map type inspect IN-2-OUT-PMAP
R3(config-pmap)#class type inspect INTERNAL-CLASS-MAP
R3(config-pmap-c)#inspect
o Create a zone pair named IN-2-OUT-ZPAIR that identifies IN-
ZONE as the source zone and OUT-ZONE as the destination zone.
R3(config)#zone-pair security IN-2-OUT-ZPAIR source IN-ZONE
destination OUT-ZONE
o Specify that the IN-2-OUT-PMAP policy map is to be used to
inspect traffic between the two zones.
R3(config-sec-zone-pair)#service-policy type inspect IN-
2-OUT-PMAP
o Assign Fa0/1 as an IN-ZONE member and S0/0/1 as an OUT-ZONE
member.
R3(config)#interface fastEthernet 0/1
R3(config-if)#zone-member security IN-ZONE
R3(config)#interface serial 0/0/1
R3(config-if)#zone-member security OUT-ZONE
Configure an IPS on R3 using the following requirements:
Note: Within Packet Tracer, the routers already have the signature files
imported and in place. They are the default XML files in flash. For this
reason, it is not necessary to configure the public crypto key and
complete a manual import of the signature files.
o Create a directory in flash named ipsdir and set it as the
location for IPS signature storage.
R3#mkdir ipsdir
R3#dir flash
o Create an IPS rule named IPS-RULE.
R3(config)#ip ips config location flash:ipsdir
R3(config)#ip ips name IPS-RULE
o Retire the all signature category with the retired true
command (all signatures within the signature release).
R3(config)#ip ips signature-category
R3(config-ips-category)#category all
R3(config-ips-category-action)#retired true
o Unretire the IOS_IPS Basic category with the retired false
command.
R3(config-ips-category)#category ios_ips basic
R3(config-ips-category-action)#retired false
o Apply the rule inbound on the S0/0/1 interface.
R3(config)#interface serial 0/0/1
R3(config-if)#ip ips IPS-RULE in

Configure ASA Basic Security and Firewall Settings
Configure VLAN interfaces with the following settings:
o For the VLAN 1 interface, configure the addressing to use
192.168.10.1/24.
ciscoasa(config)#interface vlan 1
ciscoasa(config-if)#ip address 192.168.10.1
255.255.255.0
o For the VLAN 2 interface, remove the default DHCP setting and
configure the addressing to use 209.165.200.234/29.
ciscoasa(config)#interface vlan 2
ciscoasa(config-if)#no ip address dhcp
ciscoasa(config-if)#ip address
209.165.200.234 255.255.255.248
Configure hostname, domain name, enable password, and Telnet console
password using the following settings:
o The ASA hostname is CCNAS-ASA.
ciscoasa(config)#hostname CCNAS-ASA
o The domain name is ccnasecurity.com.
CCNAS-ASA(config)#domain-name ccnasecurity.com
o The enable mode password is ciscoenapa55.
CCNAS-ASA(config)#enable password ciscoenapa55
Create a user and configure AAA to use the local database for remote
authentication.
o Create a local user account of Admin01 with a secret password
of Admin01pa55 and a privilege level of 15.
CCNAS-ASA(config)#username Admin01 password
Admin01pa55
o Configure a local user account named admin with the password
adminpa55. Do not use the encrypted attribute.
CCNAS-ASA(config)#username admin password
adminpa55
o Configure AAA to use the local ASA database for Telnet and SSH
user authentication.
CCNAS-ASA(config)#aaa authentication ssh console
LOCAL
CCNAS-ASA(config)#aaa authentication telnet
console LOCAL
Configure Telnet for local ASA console access and SSH for remote ASA
console access.
o Allow Telnet access from the inside 192.168.10.0/24 network
with a timeout of 10 minutes.
CCNAS-ASA(config)#telnet 192.168.10.0
255.255.255.0 inside
CCNAS-ASA(config)#telnet timeout 10
o Allow SSH access from the outside host 172.30.3.3 with a
timeout of 10 minutes.
CCNAS-ASA(config)#ssh 172.30.3.3 255.255.255.0
outside
CCNAS-ASA(config)#ssh timeout 10
Configure the ASA as a DHCP server using the following settings:
o Assign IP addresses to inside DHCP clients from 192.168.10.5
to 192.168.10.30.
CCNAS-ASA(config)#dhcpd address 192.168.10.5-
192.168.10.30 inside
o Enable DHCP to listen for DHCP client requests.
CCNAS-ASA(config)#dhcpd enable inside
Configure static routing and NAT.
o Create a static default route to the next hop router (R1) IP
address.
CCNAS-ASA(config)#route outside 209.165.200.232
255.255.255.248 209.165.200.233
o Create a network object named inside-net and assign attributes
to it using the subnet and nat commands.
CCNAS-ASA(config)#object network inside-net
CCNAS-ASA(config-network-object)#subnet
192.168.10.0 255.255.255.0
CCNAS-ASA(config-network-object)#nat
(inside,outside) dynamic interface
o Create a dynamic NAT translation to the outside interface.
Modify the Cisco Modular Policy Framework (MPF) on the ASA using the
following settings:
o Configure class-map inspection_default to match default-
inspection-traffic, and then exit to global configuration mode.
CCNAS-ASA(config)#class-map instection_default
CCNAS-ASA(config-cmap)#match default-
inspection-traffic
o Configure the policy-map list, global_policy. Enter the class
inspection_default and enter the command to inspect icmp. Then exit to
global config mode.
CCNAS-ASA(config)#policy-map global_policy
CCNAS-ASA(config-pmap)#class
instection_default
CCNAS-ASA(config-pmap-c)#inspect icmp
o Configure the MPF service-policy to make the global_policy
apply globally.
CCNAS-ASA(config)#service-policy global_policy
global
